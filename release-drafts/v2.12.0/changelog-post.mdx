## v2.12.0 is live

Firecrawl v2.12.0 lands a `search` monitor target that runs SERP queries on a schedule and alerts on new or materially-developed results (with a verifier → criteria → skeptic → reviewer → event-resolver judge stack and flat, deterministic billing), a deterministic `menu` scrape format gated behind `menuBeta`, and a hosted parse upload flow so MCP and remote tools can call `/v2/parse` on local files without exposing API keys. It also adds a `SELF_HOSTED_DOMAIN` env var to fix response URLs behind reverse proxies, upgrades the API to Express 5.2 and the build compiler to TypeScript 7 RC, ships NuQ FoundationDB metrics under a new admin endpoint, and finishes the migration to Autumn as the single source of truth for credit accounting across v0, v1, and v2.

### Highlights

- **`search` monitor target** — Added a first-class `search` monitor target accepting `queries[]`, `searchWindow`, `includeDomains[]`, `excludeDomains[]`, and `maxResults`, with `raw` (no LLM) and `deep` (LLM judge) depths derived from `judgeEnabled`, a `material_dev` alert mode that re-alerts on material developments of a known event, and stable opaque event keys to prevent label-drift duplicates. Exposed across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs with a `MonitorSearchTargetResult` type.
- **Search monitor billing** — Search monitors now bill at a flat per-check rate: `SEARCH_CREDITS` is charged `ceil(totalResults / 10) * 2`, judged results are charged 1 credit per result (down from 5), and `raw` mode pays nothing for the judge. Deep-mode per-page scrapes flow through the shared scrape pipeline as queued jobs billed under `CREDITS`, not double-counted on the search check.
- **`menu` scrape format** — A new `formats: ["menu"]` returns deterministic structured restaurant-menu data as `document.menu`, mirroring the existing `product` format end-to-end. Beta-gated via the `menuBeta` team flag with an `MENU_EXTRACTION_SERVICE_URL` self-host kill-switch, available across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **Hosted `/v2/parse` upload flow** — `POST /v2/parse/upload-url` mints a short-lived, team-scoped `uploadRef` plus signed storage upload parameters; the client uploads bytes directly to GCS; then `POST /v2/parse` accepts JSON with `uploadRef` and parses normally. Refs are HMAC-signed with a 10-minute expiry and bound to the team, object path, filename, content type, and max size, so MCP clients can parse local files without exposing an API key to the model or shell.
- **`SELF_HOSTED_DOMAIN` env var** — Self-hosted instances behind a reverse proxy can now set `SELF_HOSTED_DOMAIN` (or send `X-Forwarded-Host`) so crawl status URLs, batch-scrape URLs, parse-upload links, and pagination `next` links use the externally-reachable host instead of the internal bind address.
- **Express 5 and TypeScript 7 RC** — The API and worker now run on Express 5.2.1 with `express-ws` reapplied to the v1/v2 routers, and the API build compiler is `typescript@7.0.1-rc` for roughly 3× faster clean builds (median ~1.84 s vs ~5.60 s on TS 5.9.2). `knip` continues to run on TS 5 through a wrapper to avoid runtime API breaks.
- **NuQ FoundationDB metrics** — New per-status FDB queue gauges (`queued`, `active`, `backlog`, `completed`, `failed`) and a `firecrawl_nuq_fdb_pending_jobs` worker-load gauge, served from a new `GET /admin/<BULL_AUTH_KEY>/nuq-fdb-metrics` Prometheus endpoint for FDB-based queue autoscaling.
- **Search highlights `/highlight` service** — The `highlightsBeta` `/v2/search` snippet rendering path now delegates to the new `/highlight` model service: the full markdown and query are POSTed and the reassembled markdown is returned as the snippet. The legacy client-side sentence-split/budget pipeline is gone.
- **Autumn is the single source of truth for credits** — Removed the last ACUC-based credit accounting paths from the v0/v1/v2 hot path. v0 `scrape`, `crawl`, and `search` now gate on `autumnService.checkCredits`, the unverified agent-key 50-credit cap reads `getTeamBalance().usage` from Autumn, and the dead `checkTeamCredits` helpers have been removed. ACUC concurrency RPCs now call `v48`, giving single-team orgs `GREATEST(override, plan)` and pay-as-you-go orgs a concurrency floor of 5.
- **Team-scoped historical credit and token usage** — `/v1` and `/v2` `team/credit-usage/historical` and `team/token-usage/historical` now return only usage tracked against the API key's team, with the previous customer-level fallback removed.
- **`proxy: enhanced` / `stealth`** — Fixed `proxy: enhanced` and `proxy: stealth` scrapes silently routing through non-stealth engines on the cloud — the fallback list is now restricted to stealth-capable engines when stealth is requested.
- **Playwright cookie redirect fix** — Playwright-engine scrapes now seed the `Cookie` header into the browser context via `context.addCookies()` scoped to the registrable domain, so authenticated pages that 302 (including cross-subdomain hops) no longer come back as the login page.
- **OpenAI fallback for scrape query answers** — `format: "question"` and freeform `query` scrapes now fall back to `gpt-4o-mini` between Google Gemini and Vertex Gemini, so `document.answer` keeps populating in OpenAI-only environments when Gemini is throttled and Vertex credentials are missing.
- **Screenshot cache cutoff** — Screenshot and full-page screenshot cache entries created before `2026-06-24T20:00:00Z` are no longer reused — affected scrapes force a fresh capture instead of returning broken cached URLs.
- **CVE patches** — Resolved multiple CVEs by upgrading `multer` to `2.2.0` (GHSA-3p4h-7m6x-2hcm, GHSA-72gw-mp4g-v24j) and `undici` to `7.28.0` with a pnpm override locking it major-bound (GHSA-35p6-xmwp-9g52, GHSA-g8m3-5g58-fq7m, GHSA-hm92-r4w5-c3mj, GHSA-p88m-4jfj-68fv, GHSA-pr7r-676h-xcf6, GHSA-vmh5-mc38-953g, GHSA-vxpw-j846-p89q), bumping `artillery` in `apps/test-suite`, and removing the malicious `serpapi-python==0.1.5` pin from the GPT-4.1 company researcher example in favor of `google-search-results`.
